# 🛡️ PC Health & Security Dashboard

---

## English

### Description

A public SaaS product for monitoring, securing, and protecting Windows PCs. Any user can sign up with Google, connect their PC using the lightweight local agent, and view their security dashboard from any browser — fully isolated per user.

AI-powered recommendations are generated by Claude (Anthropic) via Firebase Functions — no API keys ever touch the browser. Each user's data is strictly isolated under their own Firestore path.

---

### Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                        WEB BROWSER                          │
│              React App (Firebase Hosting)                   │
│   Login  ──►  Dashboard  ──►  Report + AI Recommendations  │
└───────────────────────┬─────────────────────────────────────┘
                        │ Firestore (real-time sync)
          ┌─────────────┴──────────────┐
          │      Firebase Cloud        │
          │  ┌─────────────────────┐   │
          │  │  Firebase Auth      │   │  Google Sign-In only
          │  │  (Google Sign-In)   │   │
          │  └─────────────────────┘   │
          │  ┌─────────────────────┐   │
          │  │  Firestore          │   │  scans, users, agents,
          │  │  (database)         │   │  pending_actions, logs
          │  └─────────────────────┘   │
          │  ┌─────────────────────┐   │
          │  │  Firebase Functions  │  │  Claude API calls
          │  │  (Node.js)          │   │  Business logic
          │  └─────────────────────┘   │
          └─────────────┬──────────────┘
                        │ Firestore (reads pending_actions, writes scan results)
┌───────────────────────┴─────────────────────────────────────┐
│                    WINDOWS PC (local)                        │
│              Python Agent (runs as .exe)                     │
│  Scanner  ──►  Backup  ──►  Cleaner  ──►  Privacy Scanner   │
└─────────────────────────────────────────────────────────────┘
```

**Data flow:**
1. Agent scans PC → writes results to `scans/{scanId}` in Firestore
2. `onScanCreated` Function triggers → calls Claude API → writes AI recommendations back to scan doc
3. Web app reads scan + recommendations from Firestore → displays dashboard
4. User requests cleanup → web app writes to `pending_actions` → agent polls and executes safely

### Installation

#### Prerequisites
- Node.js 20+
- Python 3.11+
- Firebase CLI: `npm install -g firebase-tools`
- A Firebase project with Firestore, Auth, Hosting, and Functions enabled

---

#### Web App Setup

```bash
cd web-app
cp .env.example .env
# Fill in your Firebase config values in .env

npm install
npm start          # development
npm run build      # production build
```

Deploy to Firebase Hosting:
```bash
firebase deploy --only hosting
```

---

#### Firebase Functions Setup

Set your Anthropic API key as a Firebase secret:
```bash
firebase functions:secrets:set ANTHROPIC_API_KEY
```

Deploy functions:
```bash
firebase deploy --only functions
```

---

#### Local Agent Setup (Windows PC)

1. Download `serviceAccountKey.json` from:
   Firebase Console → Project Settings → Service Accounts → Generate New Private Key

2. Set up the agent:
```bash
cd agent
pip install -r requirements.txt
cp .env.example .env
# Edit .env:
#   FIREBASE_CRED_PATH=serviceAccountKey.json
#   USER_ID=<your Firebase UID from the web app after first sign-in>
```

3. Run the agent:
```bash
python main.py
```

4. Build as Windows .exe (optional):
```bash
pyinstaller --onefile --name pc-security-agent main.py
```

---

#### Firestore Setup

Deploy security rules and indexes:
```bash
firebase deploy --only firestore
```

Enable Google Sign-In:
Firebase Console → Authentication → Sign-in method → Google → Enable

### Usage Guide

1. Open the web app and sign in with your Google account
2. Download and run the local agent on the Windows PC you want to monitor
3. The agent automatically sends a heartbeat every 30 seconds — the dashboard shows "Agent Connected" when live
4. The agent runs an initial scan on startup and then every 60 minutes
5. Open the **Reports** page to see scan results + AI recommendations
6. Use **Preview** mode to review what would be cleaned before any action
7. Toggle off Preview mode to enable live cleanup (auto backup always created first)
8. Use **Undo** to restore files from backup if needed

---

### Safety Principles

| Principle | Implementation |
|---|---|
| **Read-only by default** | `READ_ONLY_MODE = True` in agent; all mutations require explicit opt-in |
| **Preview before deletion** | Report page shows all items in Preview mode; no changes made until confirmed |
| **Auto backup before cleanup** | `BackupManager.create_backup()` is always called before `Cleaner.clean_items()` |
| **Full undo capability** | Backup ZIP restored via `BackupManager.restore_backup()`; undo button in UI |
| **Protected paths** | `C:\Windows`, `C:\Program Files` and system roots are blocked in cleaner |
| **Recycle bin first** | `send2trash` moves files to recycle bin instead of permanent deletion on Windows |
| **Detailed audit log** | Every action (clean, backup, undo) written to `action_log` collection in Firestore |

---

### Auth & Data Rules

- **Google Sign-In only** — no passwords, no email/password auth
- Users can only access their own scan data (enforced in `firestore.rules`)
- Agent writes to Firestore via service account (Admin SDK) — not accessible from browser
- User schema includes `plan` field (`"free"` / `"pro"`) for future paid tiers

- Full **Hebrew** and **English** support
- Hebrew renders in **RTL** (`dir="rtl"`), English in **LTR**
- Language toggle button in all pages (persisted to `localStorage`)
- AI recommendations generated in the user's preferred language

## עברית

### תיאור הפרויקט

מערכת היברידית לניטור, ניקוי ואבטחת מחשב Windows. לוח הבקרה (React + Firebase) נגיש מכל דפדפן, בעוד שסוכן Python קל-משקל פועל מקומית על המחשב הנסרק ומתקשר עם Firebase בזמן אמת.

המלצות מופקות על ידי Claude (Anthropic) דרך Firebase Functions — מפתחות API לא נחשפים לדפדפן.

### ארכיטקטורה

- **אפליקציית ווב** (React): גישה מכל דפדפן, מציגה תוצאות סריקה והמלצות AI
- **סוכן מקומי** (Python .exe): פועל על המחשב הנסרק, שולח נתונים ל-Firestore
- **Firebase Functions**: מטפל בקריאות ל-Claude API ובלוגיקה עסקית
- **Firestore**: מאגר נתוני הסריקה, משתמשים ויומן פעולות

### עקרונות בטיחות

| עיקרון | יישום |
|---|---|
| **מצב קריאה בלבד כברירת מחדל** | כל שינוי דורש הסכמה מפורשת |
| **תצוגה מקדימה לפני מחיקה** | מציג את כל הפריטים לפני כל פעולה |
| **גיבוי אוטומטי לפני ניקוי** | גיבוי ZIP נוצר תמיד לפני מחיקה |
| **ביטול מלא** | שחזור מהגיבוי בלחיצה אחת |
| **נתיבים מוגנים** | `C:\Windows`, `C:\Program Files` חסומים |
| **יומן פעולות מפורט** | כל פעולה נרשמת ב-Firestore |
